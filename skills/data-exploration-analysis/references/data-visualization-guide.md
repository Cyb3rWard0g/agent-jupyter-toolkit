# Data Visualization Guide

Use visualizations to reveal patterns and communicate findings.

## Principles

Visualizations should clarify data patterns, not decorate them.

Prefer simple and interpretable charts.

---

## Common visualization types

### Bar charts

Useful for comparing categorical counts.

Examples:
- logon types
- events per user
- top source IPs

---

### Time series charts

Useful for understanding temporal patterns.

Examples:
- events over time
- spikes in activity
- login frequency

---

### Histograms

Useful for understanding distributions.

Examples:
- event frequency
- duration distributions

---

### Heatmaps

Useful for identifying concentration patterns.

Examples:
- activity by hour and day
- event density across hosts

---

## Visualization guidelines

Prefer libraries that generate static images.

Recommended:
- matplotlib
- seaborn

Avoid overly complex or interactive visualizations unless they add analytical value.

Every visualization should answer a specific analytical question.
